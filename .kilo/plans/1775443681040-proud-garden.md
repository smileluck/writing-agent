# 分析报告：temp/prompts 对现有 Skills/Agents 的参考与优化空间

## 一、全局判断

`temp/prompts/` 是一个**网文（起点中文网长篇小说）创作系统**的 Prompt 集，包含 22 个文件，覆盖从概念构思到章节生成的完整流水线。

现有 `.kilo/` 体系是一个**微信公众号/自媒体文章写作系统**，包含 3 个 Skills + 18 个 Agents。

**核心差异**：两者面向的创作类型不同（长篇小说 vs 自媒体文章），但 `temp/prompts` 中有大量高质量的技术手段可以反哺现有体系。

---

## 二、逐项对比与优化建议

### 2.1 可直接借鉴的高价值技术（推荐吸收）

| # | temp/prompts 来源 | 现有对应 Agent/Skill | 可借鉴内容 | 优先级 |
|---|---|---|---|---|
| **1** | `writer_persona.md` → 反AI检测规则（行12-32） | `humanizer.md` + `writing-executor.md` | **人类写作特征清单极精炼**：句式长短不一、不完整句子、口语化、口头禅、思维跳跃、段落长度随内容变化。现有规则偏"禁令式"（不许用XX词），缺少正面"必须做到"的清单。建议整合为 `humanizer.md` 的补充章节。 | 🔴 高 |
| **2** | `writing_v2.md` → 严格有限视角（行9-12） | `writing-executor.md` | **全知视角禁令**：禁止"与此同时""另一边""殊不知""他并不知道"。自媒体文章虽非小说，但这个规则可以有效防止AI常用的"上帝视角评述"（如"在当今社会，人们普遍认为..."）。建议在 `writing-executor.md` 的反AI规则中加入"禁止全知旁白"一条。 | 🟡 中 |
| **3** | `optimize_dialogue.md` → 潜台词丰富化 + 非语言元素 | `concretizer.md` | **对话优化技术**：潜台词设计（"你还好吗？" → 可能是"你还爱我吗？"）、非语言元素（转身、低头、握拳）。现有 `concretizer.md` 只做类比/画面/行动三维度，可以加入"对话潜台词设计"作为第四维度，特别适合人物引述和场景描写。 | 🟡 中 |
| **4** | `optimize_environment.md` → 五感全开 + 环境氛围参考表 | `concretizer.md` + `writing-executor.md` | **五感描写法 + 情绪-环境对照表**（行56-62）。这张表极其实用：为每种情绪目标给出了光线、声音、气味、温度、空间感的具体参数。建议作为 `concretizer.md` 的 reference 文件或在 `writing-executor.md` 的写作提示中引用。 | 🔴 高 |
| **5** | `optimize_psychology.md` → 心理活动五技巧 | `writing-executor.md` + `humanizer.md` | **五种心理描写技巧**：内心独白、意识流、身体反应、记忆闪回、自我对话。配合"情绪复杂性"原则（愤怒里有委屈、快乐里有不安）。这恰好解决了现有体系"只说不要总结，没说该怎么写内心"的问题。建议在 `writing-executor.md` 的 Step 4 中增加"心理描写技巧"小节。 | 🔴 高 |
| **6** | `optimize_rhythm.md` → 节奏韵律系统 | `writing-executor.md` | **标点符号个性化指南**（行42-63）：省略号表示思绪飘散、破折号表示突然转念、感叹号克制使用等。现有体系只有"节奏检查"（打断相同长度句子），但缺少标点层面的指导。建议整合进 `writing-executor.md`。 | 🟡 中 |
| **7** | `character_dna_guide.md` → 八维度角色构建 | `风格建模/SKILL.md`（references） | **角色DNA八维度**：童年创伤、核心恐惧、内心渴望、说话习惯、身体语言、思维模式、决策方式、隐藏秘密。虽然自媒体文章不塑造角色，但**风格建模的15维度体系可以参考此方法论**，特别是"说话习惯"和"思维模式"维度，可以细化风格建模的粒度。 | 🟢 低 |
| **8** | `concept.md` → 动态对话引导 + 选择题启发 | `writing-clarifier.md` | **信息清单驱动的对话流程**：temp 的 `concept.md` 用内部 Checklist 驱动多轮问答，每次提供 6+ 选项。现有 `writing-clarifier.md` 是一次性的信息收集。可以在 Clarifier 中增加"为用户提供选项灵感"的机制，特别是风格选择和读者画像确认环节。 | 🟡 中 |

### 2.3 已有优势（现有体系领先于 temp/prompts 的地方）

| 维度 | 现有体系 | temp/prompts |
|---|---|---|
| **AI味检测** | 三层20+项系统化检测（editor-review.md），50分制自评（humanizer.md） | 只有简单禁令（"禁止AI套话"），无系统化检测 |
| **社交传播设计** | 八大社交货币 + Share Map + 截图点设计（empathy-designer.md） | 只有"读者代入感场景"7种，较浅 |
| **立场设计** | 独立的 position-engine，六步强制推演 | 无对应模块 |
| **标题设计** | 15种爆款公式库 + 7项自检清单（title-designer.md） | 只有概念对话中顺带生成标题 |
| **工作流编排** | 18个Agent + 13阶段完整SOP + 强制中断点 | 无统一编排，各Prompt独立运行 |

## 四、总结

| 类别 | 数量 | 说明 |
|---|---|---|
| 可直接借鉴的高价值技术 | 8 项 | 其中 3 项高优先级、3 项中优先级、2 项低优先级 |
| 暂不适用但值得关注 | 9 项 | 面向网文场景，与自媒体文章不匹配 |
| 现有体系已有优势 | 5 个维度 | AI味检测、社交传播、立场设计、标题设计、工作流编排 |

**核心结论**：temp/prompts 最大的价值在于**微观写作技术**（五感描写、心理描写、节奏标点、对话潜台词），这些恰好是现有体系在"禁止什么"之外缺少的"应该怎么写"的正向指导。建议优先吸收阶段一的 4 项改动。
